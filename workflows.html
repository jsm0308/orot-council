핵심 워크플로우 (v3)

6개의 핵심 작업. 모두 한국어 자연어로 트리거된다.

Ingest (문서 주입) "읽어서 위키에 반영해줘"

소스 문서를 읽고 핵심 내용을 추출한다. raw/ 디렉토리 또는 2_Sources/에서 선택. 소스 하나당 보통 5~15페이지를 수정하며 prose 관계와 모순 탐지도 함께 수행.

1소스 읽기 (PDF, DOCX, PPTX, HTML, TXT)
2요약 + kind/topics/subject 결정 + 사용자 확인
3source-record 페이지 생성
4concept/entity 페이지 업데이트 + prose 관계
5모순 탐지 + index.md + log.md 갱신

v3 신규: 소스는 source-record로 저장, concept과 명확히 구분. prose 관계 자동 생성.

Save (저장) "저장해줘"

현재 대화 응답을 위키 페이지로 저장. kind/form/subject/topics/source-types를 LLM이 자동 분류하고 prose 관계를 추가한다.

1kind + form + topics + subject 자동 분류
2v3 프론트매터 생성
3prose 관계 추천 (builds on [[x]])
4index.md + log.md 갱신

v2 -> v3: related 필드 제거, prose 관계로 대체. source_count -> source-types.

Synapse (결정 페이지) -- 신규 "결정 기록해줘" "/synapse"

대화형으로 결정 페이지 작성을 유도. 13개 섹션을 하나씩 물어보며, 빈 섹션은 자동 삭제. kind=decision으로 저장.

1Context -- 배경 질문
2Problem -> Decision -> Alternatives
3Rationale -> Mechanism -> Outcome
4Failure mode -> Iteration -> Invariant
5Reusability -> Related -> Next action
6decision-YYYY-MM-DD-slug.md로 저장 + prose 관계
Search / Query (검색 + 질문) "검색해줘 <검색어>"

qmd + 벡터 검색으로 위키를 검색. 관련 페이지를 찾아 응답에 포함시키고 [[링크]]로 인용. 응답 후 저장 제안.

Lint (건강검진) -- 수동 실행 "점검해줘"

9종 검사 수행. 자동 실행되지 않음. 리포트 저장 + manifest/dependencies 자동 갱신.

v3 신규: schema validation, domains cache, subject path, topic vocabulary, prose 관계 검증.

Promote (아카이브 이전) "마이그레이션해줘"

v2 아카이브를 v3 위키로 승격. scripts/migrate-v2-to-v3.py 스크립트 실행. category->kind, tags->topics+subject, related->prose 관계 변환.

Lint 검사 항목 (9종)

20회 이상 인제스트했거나 1주 이상 지났으면 실행을 권장한다.

Schema Validation 신규
kind, form, source-types, confidence 값이 유효한지 확인.
Domains Cache 신규
domains 필드가 subject 최상위와 일치하는지 확인.
Subject Path 신규
subject 경로가 subject-tree.md에 존재하는지 확인.
모순 탐지
서로 다른 페이지에서 상충되는 주장 확인.
오래된 페이지
updated가 90일 이상 지난 페이지.
고립 페이지
다른 페이지에서 링크되지 않은 페이지.
Topic Vocabulary 신규
topics.md에 없는 unfamiliar 키워드 경고.
누락된 링크
연결되어야 하지만 링크가 없는 페이지 쌍.
데이터 공백
추가 증거가 필요한 주장.
