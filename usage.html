사용 가이드

JSM Wiki 시스템을 실제로 사용하는 방법. 시작부터 명령어 레퍼런스까지.

실행 환경

권장 Cursor IDE

Cursor IDE 안에서 직접 대화. 모든 기능이 자연어로 작동.

"읽어서 위키에 반영해줘" → 인제스트

"저장해줘" → 위키 페이지로 저장

"검색해줘 attention" → 위키 검색

chat.py CLI

python chat.py로 실행. / 명령어 추가 지원.

/tone research → 연구 멘토 모드

/search 쿼리 → 위키 검색

/calendar today → 오늘 일정

모드는 자동 감지된다. "/tone research"를 입력하지 않아도 "연구 어떻게 시작해?"라고 말하면 Research Mentor로 전환된다. 모드가 바뀌면 응답 앞에 (research), (study) 같은 표시가 붙는다. 수동 전환이 필요하면 /tone {mode}를 사용한다.

빠른 시작

1 실행

Cursor IDE: 그냥 채팅 시작

rule 파일 자동 로드

CLI: python chat.py

Obsidian 연결 상태 확인

2 대화

명령어 없이 자연스럽게 말한다.

"연구 어떻게 시작해?"

→ (research) 자동 전환

"오늘 운동 뭐할까?"

→ (fitness) 자동 전환

3 응답

코치가 현재 모드에 맞춰 응답한다.

Research: 관심사 파악 → 논문 추천 → 액션 아이템

Study: 개념 설명 + 위키 [[링크]] 인용

일반적인 연구 세션 흐름

1단계
PDF를 2_Sources/papers/에 넣는다
2단계
"읽어서 위키에 반영해줘" → 파일 선택 → 핵심 내용 추출
3단계
"무엇을 강조할까?" → 한 개념 골라 깊이 파기
4단계
위키 페이지 생성 (프론트매터 + 크로스레퍼런스)
5단계
멘토에게 질문: "이 개념 다른 논문에선?"
6단계
세션 종료. 1-3일 액션 아이템 1-3개
