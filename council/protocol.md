# 아키텍트 소집 프로토콜 (Mode B)

> Mode B(Architecture Master) 발동 시 병렬 렌즈를 소집하는 절차.

## §1 소집 조건

- 기술 스택·아키텍처·DB·프레임워크 등 **비가역적이거나 비용이 큰 결정**일 때.
- 작은 설계 질문(패키지 하나 고르기 등)은 에이전트가 직접 처리. 과소집 금지.

## §2 입력 형식

결정을 아래 4줄로 정리하여 소집:
```
결정: <무엇을 결정해야 하는가>
선택지: <A, B, C ...>
제약: <비용/시간/기술 제약>
좋은 결정: <3개월 뒤 "잘 골랐다"라고 느낄 조건>
```

## §3 출력 계약

각 렌즈는 **정확히 4블록**만 출력:
1. **핵심 진단** — 해당 렌즈 관점에서 본 핵심 1~3개
2. **가장 치명적인 지점** — 단 하나
3. **구체적 권고** — 어떻게 할지 (실행 가능하게)
4. **판정** — `GO / GO-IF(<조건>) / NO-GO` + 한 줄 이유

## §4 취합

에이전트가 렌즈 결과를 다수결이 아닌 **논거의 무게**로 취합, 한 장 보고:
```
[한 줄 결론]
[렌즈별 판정 요약 표]
[핵심 분기점 — 의견이 갈리는 지점]
[권고 + 사용자가 결정할 지점]
```
